API Endpoints

GET /

Get API Information

Returns API version and available endpoints

Property Type Description
version string Current API version
endpoints array Available endpoint paths
POST /download

Download MP3

Download a song as MP3 file

Note: Conversion can take 10-30 seconds depending on video length
Property Type Description
url string YouTube video URL (required)
bitrate string Audio bitrate: 128k, 192k, 256k, 320k (default: 192k)
format string Format type: mp3, m4a (default: mp3)
GET /status/:id

Check Conversion Status

Get the status of a download task

Property Type Description
id string Task ID from download endpoint
status string pending, processing, completed, error
GET /file/:id

Retrieve MP3 File

Download the converted MP3 file

Property Type Description
id string Task ID from download endpoint
filename string Custom filename (optional)
POST /batch

Batch Download

Convert multiple videos in one request

Property Type Description
urls array Array of YouTube URLs
bitrate string Audio bitrate for all files

Check Status

// Poll for completion
async function checkStatus(taskId) {
    let status = 'processing';
    while (status === 'processing') {
        const response = await fetch(`/status/${taskId}`);
        const data = await response.json();
        status = data.status;
        
        if (status === 'completed') {
            console.log('Download ready!');
            window.location.href = `/file/${taskId}`;
        } else if (status === 'error') {
            console.error('Conversion failed:', data.error);
        }
        
        // Wait 2 seconds before checking again
        await new Promise(r => setTimeout(r, 2000));
    }
}

Error Handling

Error Response Format

{
    "error": true,
    "code": "INVALID_URL",
    "message": "The provided URL is not a valid YouTube link",
    "statusCode": 400
}

Common Errors

Code HTTP Status Description Solution
INVALID_URL 400 URL is not a valid YouTube link Verify the YouTube URL format
VIDEO_NOT_FOUND 404 Video doesn't exist or is deleted Check if video is still available
RESTRICTED 403 Video is restricted or region-locked Try a different video
FILE_TOO_LARGE 413 Video exceeds 50MB limit Use shorter videos or lower bitrate
CONVERSION_FAILED 500 Backend conversion error Retry the request
RATE_LIMIT_EXCEEDED 429 Too many requests Wait before making new requests

Error Handling Example

async function downloadMP3(url) {
    try {
        const response = await fetch('/download', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ url })
        });

        if (!response.ok) {
            const error = await response.json();
            throw new Error(`${error.code}: ${error.message}`);
        }

        return await response.json();
    } catch (error) {
        console.error('Error:', error.message);
        
        if (error.message.includes('INVALID_URL')) {
            alert('Please enter a valid YouTube URL');
        } else if (error.message.includes('FILE_TOO_LARGE')) {
            alert('Video is too large for conversion');
        }
    }
}

Rate Limiting & Quotas

The API implements rate limiting to ensure fair usage across all clients. Rate limit information is provided in response headers.

Rate Limit Headers

Header Description
X-RateLimit-Limit Maximum requests per minute
X-RateLimit-Remaining Remaining requests in current window
X-RateLimit-Reset Unix timestamp when limit resets

Quotas

  • Free Tier: 10 conversions/hour, 5GB/month
  • Pro Tier: 100 conversions/hour, 50GB/month
  • Enterprise: Unlimited
Consider implementing exponential backoff when handling rate limit responses (429 status).

Best Practices

URL Validation

Always validate YouTube URLs before sending to the API:

function isValidYouTubeURL(url) {
    const patterns = [
        /(?:youtube\.com\/watch\?v=|youtu\.be\/)([a-zA-Z0-9_-]{11})/,
        /(?:youtube\.com\/embed\/)([a-zA-Z0-9_-]{11})/
    ];
    return patterns.some(p => p.test(url));
}

Polling Strategy

  • Use exponential backoff when polling status
  • Start with 1-2 second intervals
  • Cap polling at 30 second intervals
  • Set maximum timeout at 5 minutes

Error Recovery

  • Implement retry logic with exponential backoff
  • Don't retry on 4xx errors (client errors)
  • Retry 5xx errors up to 3 times
  • Log all errors for debugging
Always cache results and reuse downloads when possible to reduce API calls.

Frequently Asked Questions

What video formats are supported?

Currently supports YouTube videos only. Other platforms may not work due to copyright protection.

What's the maximum file size?

Maximum file size is 50MB. Larger videos will return a FILE_TOO_LARGE error.

How long does download take?

Typically 10-30 seconds for songs. Longer videos may take up to 60 seconds.

Is commercial use allowed?

Not recommended without proper licensing from YouTube. Check copyright laws in your jurisdiction.

Can I download private videos?

No, only public and unlisted videos can be downloaded.

What's the difference between bitrates?

Higher bitrates (256k, 320k) provide better audio quality but larger file sizes. 192k is recommended for most use cases.
